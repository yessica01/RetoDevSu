# RetoDevSu

Descripción: El proyecto de automatización de compras DevSu está construido para realizar las cobertura
de pruebas funcionales y E2E para el aplicativo de compra web:Saucedemo y la cobertura de pruebas funcionales
para el servicio API de usuario de la tienda Petstore dadas por el negocio.

## Estado del Proyecto
Este proyecto se encuentra en una etapa de desarrollo abierta a cambios por nuevos requerimientos y/o ajustes por parte del equipo de desarrollo.
Actualmente es un producto mínimo.

Se entrega con deuda técnica de code smell por motivos de tiempo limitado para el desarrollo optimo del aplicativo.
El compromiso es disminuir la deuda técnica durante el sprint.

## Instalación
Para ejecutar este robot de automatización se encuentran disponibles los runners de ejecución correspondientes a cada funcionalidad.
Estos runners están divididos por carpetas (frontend, backend, e2e), los escenarios que tiene la anotación @EscenarioCritico son los escenarios
más importantes para el negocio

## Uso
Al ejecutar cada runner desde un IDE como intellij, se genera el reporte de pruebas en consola y en la carpeta Target
